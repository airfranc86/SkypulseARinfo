# SkyPulse AR — Design Context

## Design Context

### Users
Argentinos que quieren tomar decisiones concretas basadas en el estado del tiempo y fenómenos naturales: si salir, si tender ropa, si es seguro viajar, si hay actividad sísmica o volcánica cerca. Consultan en el celular, en contextos cotidianos. No son meteorólogos — necesitan datos accionables en segundos.

### Brand Personality
**Nocturno. Preciso. Austero.**
La app vive en la noche azul argentina. El dorado es la única calidez en un universo frío de datos. No es alarmista — informa con calma y exactitud. La urgencia se expresa con color, no con signos de exclamación.

### Aesthetic Direction
- **Tema**: Dark-only. `#060d1a` azul nocturno profundo + `#c8a84b` dorado accent.
- **Tipografía existente**: Playfair Display (display/serif) + DM Sans (UI). Son parte del design system establecido — no modificar.
- **Tokens semánticos de peligro**: `#3ecf7a` (safe) → `#f0a030` (watch) → `#e05545` (warn) → `#ff3333` (crit). Mapean directamente a verde/amarillo/naranja/rojo.
- **Componentes**: Pills de color para nav items, ElectricBorder para tarjetas destacadas, FadeContent para entradas, FallingText decorativo.
- **Textura**: Starfield background fijo + Threads animados en desktop.

### Design Principles
1. **Acción sobre descripción** — cada pantalla responde UNA pregunta con UNA acción concreta.
2. **Color como semáforo** — el sistema de colores de peligro es el lenguaje principal de urgencia. No usar texto alarmista.
3. **Jerarquía por datos, no por decoración** — el volcán con mayor riesgo se ve más grande/prominente, no porque tenga más bordes.
4. **Sin side-stripe borders** — usar fondos tintados + bordes completos para alertas.
5. **Consistencia con el sistema existente** — nuevas secciones deben usar los tokens ya definidos (`--color-warn`, `--color-crit`, `--color-safe`, etc.).
